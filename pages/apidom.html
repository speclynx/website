SpecLynx ApiDOM

Open Source

Semantic parser for API descriptions

ApiDOM provides a single, unifying structure for describing APIs across different description languages (OpenAPI, AsyncAPI, JSON Schema, Arazzo) and serialization formats (JSON, YAML).

View on GitHub Try the Playground

Rather than handling each format individually, ApiDOM offers one consistent structure for all API specifications. Like XML's DOM, it provides query capabilities that decouple code from specific document structures.

Key Features

✓ Semantic parsing
✓ Format unification
✓ Queryable & traversable
✓ Annotation layer
✓ Cross-format conversion
✓ Reference resolution

Core Concepts

Elements

ApiDOM structures consist of semantic elements representing API components. Each element carries meaning about its role in the API specification, from resource definitions to HTTP request examples. 

import { ObjectElement, StringElement } from '@speclynx/apidom-core';

// Create semantic elements
const title = new StringElement('My API');
const info = new ObjectElement({ title });

// Access element metadata
console.log(title.element); // 'string'
console.log(info.element);  // 'object'

Processing Stages

ApiDOM operates through three processing stages:

1. Parse Stage

Converts JSON/YAML strings to ApiDOM via lexical and syntactic analysis. Produces a generic ApiDOM structure.

2. Refract Stage

Transforms generic ApiDOM into semantic structures specific to OpenAPI, AsyncAPI, or other specifications.

3. Generate Stage

Produces JSON/YAML documents from ApiDOM. Note: style information is not preserved during generation.

import { parse } from '@speclynx/apidom-parser-adapter-openapi-yaml-3-1';

// Parse stage: YAML string -> ApiDOM
const parseResult = await parse(`
openapi: "3.1.0"
info:
  title: My API
  version: 1.0.0
`);

// Refract stage happens automatically with semantic parsers
console.log(parseResult.api.element); // 'openApi3_1'

Annotations

ApiDOM expands standard JSON/YAML with semantic metadata while maintaining structural equivalence. This "refracting" enables cross-format conversion and semantic navigation. 

import { StringElement } from '@speclynx/apidom-core';

const element = new StringElement('value');

// Add metadata annotations
element.title = 'My Title';
element.description = 'Describes this element';
element.setMetaProperty('custom', 'metadata');

// Original value is preserved
console.log(element.toValue()); // 'value'

Installation

ApiDOM requires Node.js >=16.14.2 <=24. Install individual packages via npm: 

# Core package
npm install @speclynx/apidom-core

# Parser adapters for your needs
npm install @speclynx/apidom-parser-adapter-openapi-yaml-3-1
npm install @speclynx/apidom-parser-adapter-asyncapi-json-2

# Utility packages
npm install @speclynx/apidom-reference
npm install @speclynx/apidom-traverse
npm install @speclynx/apidom-json-pointer

Usage

Parsing an OpenAPI Document

import { parse } from '@speclynx/apidom-parser-adapter-openapi-yaml-3-1';

const yaml = `
openapi: "3.1.0"
info:
  title: My API
  version: 1.0.0
paths:
  /users:
    get:
      summary: List users
      responses:
        '200':
          description: Success
`;

const result = await parse(yaml);
console.log(result.api.element); // 'openApi3_1'

Traversing Elements

import { traverse } from '@speclynx/apidom-traverse';

// Traverse all elements in the tree
traverse(result.api, {
  enter(path) {
    console.log('entering:', path.node.element);
  },
  leave(path) {
    console.log('leaving:', path.node.element);
  }
});

Dereferencing

import { dereference } from '@speclynx/apidom-reference';

// Resolve all $ref references
const dereferenced = await dereference(result.api, {
  resolve: { external: true }
});
