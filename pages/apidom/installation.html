Installing ApiDOM

The @speclynx/apidom-reference package is one of the main entry points for working with ApiDOM. Among other capabilities, it can parse any supported API specification into a semantic data model. 

npm install @speclynx/apidom-reference

Requires Node.js >=16.14.2. The package ships with TypeScript type definitions out of the box.

Parsing OpenAPI document into data model

Parse an OpenAPI document and access its data model to work with responses, parameters, and other components. ApiDOM has internal autodetection that applies the correct data model to the parsed document automatically. 

import { parse } from '@speclynx/apidom-reference';

const result = await parse('/path/to/openapi.json');

const { responses } = result.api.components;
const { parameters } = result.api.components;

responses.length; // number of response definitions
parameters.length; // number of parameter definitions

If you want to explicitly force a specific data model on the parsed document, use the mediaType option: 

const result = await parse('/path/to/openapi.json', {
  parse: { mediaType: 'application/openapi+json;version=3.1.2' }
});
